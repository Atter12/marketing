❌ PROBLEMA TÉCNICO CONFIRMADO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Facebook genera las URLs de medios con firmas dinámicas (oe=, oh=) 
usando JavaScript en el navegador del cliente.

Métodos probados que NO funcionan desde backend:
  ❌ Fetch directo con headers
  ❌ Proxy server
  ❌ Puppeteer (falta libnss3.so en Vercel)
  ❌ Playwright (requiere npx playwright install)

Razón: El HTML estático NO contiene las firmas, se generan 
dinámicamente cuando JavaScript se ejecuta en el cliente.

✅ SOLUCIÓN IMPLEMENTADA

1. EXTENSIÓN DE CHROME (Recomendado)
   📁 Ubicación: C:\Users\Shirley\Downloads\fb-ads-downloader
   
   Por qué funciona:
   ✅ Se ejecuta EN la página de Facebook
   ✅ Captura URLs cuando JavaScript las genera
   ✅ Tiene acceso al DOM completamente renderizado
   ✅ Intercepta peticiones de red con firmas válidas
   
   Instalación:
   1. Ir a chrome://extensions/
   2. Activar "Modo Desarrollador"
   3. "Cargar descomprimida" → Seleccionar carpeta
   4. Visitar facebook.com/ads/library
   5. ¡Botones de descarga aparecen automáticamente!

2. API WEB (Fallback limitado)
   🌐 URL: /api/facebook/download
   
   Funciona para:
   ✅ Extrae nombre de página
   ✅ Extrae texto del anuncio
   ✅ Detecta si hay video o imagen
   
   NO funciona para:
   ❌ Descargar medios (403 Bad URL hash)
   ❌ URLs sin firmas → Siempre fallan
